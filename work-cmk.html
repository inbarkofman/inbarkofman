Citi Innovation Lab 2020 – 2021

Citi Mobile Key

Replacing physical security tokens with a digital-first onboarding experience for 200K+ Citibank employees — designed to be intuitive, quick, and accessible across geographies and tech literacy levels.

User ResearchUXMobileUX Writing

The Project

One App to Replace All Tokens

Over 200K CitiBank employees use hard tokens or soft tokens (made by Safenet) to remotely connect to the Citi network. Citi Mobile Key (CMK) is an in-house app designed to replace the existing solutions and provide one unified solution to all users.

In the midst of COVID-19, remote connection became essential. This pushed the project up in priority as it produces a much better and cheaper solution for remote connection. In addition to remote connection, CMK is used to authenticate to any CitiBank in-house service.

I led the design process from research to implementation — including UX writing — working alongside a UI designer.

The Users

A Global, Cross-Cultural Workforce

Over 200K Citi employees, across several geographies, with a wide range of technological understanding. The different users will use CMK to authenticate to different Citi platforms and services, including remote connection.

User Research

Building Bottom-Up

The need for a replacement was built bottom-up. Conversations about the current remote connection solution were everywhere since COVID-19, as it was being used more frequently. Given the popularity of the conversation, users' pain-points were known in advance and taken into account from the project get-go.

To validate our assumptions, I conducted interviews with several users, then enriched the research by examining other remote connection products, collecting user feedback and learning from existing solutions.

The Problem

Three Core Pain Points

1

Dependent on IT support at setup: The setup stage was managed by the IT team, requiring hard-copy materials and a physical office visit. Users were not independent.

2

Not intuitive: The behaviour of the current solution was unclear. No instructions or indications for users — they needed frontal guidance to use the token.

3

Takes too long: The current solution required a very long time from initiation (ordering a token) to completion (connecting remotely independently).

UX Guidelines

The Design Vision

Based on research, product requirements, and the identified problems, I set the UX vision:

Intuitive, Quick & Simple

Yet fully secured and compliant with Citi legal and security standards.

Accessible

Suitable for all devices — every user should be able to install it and connect from their phone.

Friendly

Making up for the "robotic" experience of the previous solution by bringing warmth to a security app.

Inclusive

Branded-aligned UI that supports the UX writing and reflects the written information visually.

Onboarding Design

Walking Users Through

Every new user must install the app and go through the onboarding process. As a security app, this flow contains many steps and tasks — making users very active throughout. The concern was significant abandonment and potential friction.

Technical requirements covered all user tasks during onboarding: Welcome → Sign in → Password → Camera permission → Scan QR code / Type key manually → Create PIN code → Biometric permission → Completion.

Make it Simple

I divided the requirements into stages. Every stage has 1–4 steps. Users can navigate between steps, but once they complete a stage it's locked — they can't go back. This uses the security technologies: SSO password, QR code, and Biometrics.

Make it Intuitive and Indicative

A wizard stepper reflects progress to users. Every stage includes clear instructions that leave no doubt on what is required. Users receive feedback on their actions to confirm they did it correctly.

Make it Friendly

Considering the app is for everyone, I performed UX writing in a way appropriate for this broad target audience. I wanted the app to speak to humans — "translating" security terms into plain English.

Inclusive, Branded-Aligned UI

Together with the UI Designer, we set the look and feel of Citi Mobile Key. The visuals support the UX writing and reflect the written information visually.

Wireframes

From Requirements to Screens

With the onboarding flow mapped out, I moved from structure to screens. I translated each stage into low-fidelity wireframes — laying out the hierarchy, instructions, progress indicators, and feedback states for every step in the journey.

Wireframing at this level of detail allowed me to validate the flow logic with stakeholders and the development team early, before any visual design decisions were locked in.

Wireframes — All 4 Stages of the Onboarding Flow

User Testing

Validating in the Field

The onboarding flow was examined by usability testing. I led the user testing and created interactive prototypes that allowed users to interact with the flow.

I ensured the tests were executed in a way that users felt in their natural work zone — with their own corporate computer, speaking in their mother tongue language. Users received a specific task, while I took notes and recorded screens for deeper analysis. All tests were conducted remotely.

Final Design

The Finished Product

After rounds of usability testing and iteration, the final design brought together all four UX principles: intuitive step-by-step guidance, a friendly voice and tone, clear feedback at every action, and a visual identity that felt distinctly Citi — while being warmer and more approachable than any security app users had encountered before.

Final Design — Citi Mobile Key

Development

Implementation

I had sessions with the devs to gain a deeper understanding of technical requirements and limitations. These affected the UX design, and I needed to manage trade-offs to arrive at the best UX solution.

Production work was done alongside the UI designer and developers. We conducted 1:1 implementation sessions to fine-tune the design to pixel-perfect level.
